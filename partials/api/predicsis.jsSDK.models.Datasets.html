Please also note that there is no distinction between a learning dataset and a scoring dataset.

  • create(params)

    Create a dataset from a source

    Basic dataset creation 

    {
  name:       'My awesome dataset',
  source_ids: ['original_source_id'],
  header:     true,
  separator:  '\t',
  data_file:  { filename: 'source.csv' }
}

    Only name and source_ids are required.

  name:              'My awesome dataset',
  classifier_id:     $classifier_id$,
  dataset_id:        $dataset_id$,
  modalities_set_id: $modalities_set_id$,
  main_modality:     $main_modality$,
  separator:         $separator$,
  header:            $header$,
  data_file:         { filename: $name$ }
}

  • split(id, name, filename, sampling)

    Split a dataset into subsets according to smapling ratio. Note:

    • Learning subset will be named learned#dataset_name#
    • Testing subset will be named tested#dataset_name#
    • Idem for learning/testing filenames
    Parameters
    ParamTypeDetails
    idString

    Dataset id you want to split (called original dataset)

    nameString

    Name of the original dataset (used to name its subsets)

    filenameString

    Name of the original datafile (used to name its subsets's datafile)

    samplingNumber

    Examples: If you set sampling to 70, you are going to have:

    • 70% of your original dataset for learning
    • 30% of your original dataset for testing

    (default: 70)

    Returns
    Promise

    Subsets

  • update(id, changes)

    Update specified dataset You can update the following parameters:

    • {String} name
    • {Boolean} header
    • {String} separator

    If a separator is updated, this function also escape this string because there is a paradox:

    • The API requires a separator like "\t".
      To do so, the separator sent in the request must be "\t".
    • The view requires a tabulation to be able to build the preview.
      To do so, the separator given to the preview method must be "\t".
    Parameters
    ParamTypeDetails
    idString

    Id of the dictionary you want to update

    changesObject

    see above description to know parameters you are able to update

    Returns
    Promise

    Updated dataset
